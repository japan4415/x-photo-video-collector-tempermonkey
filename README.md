# x-photo-video-collector-tempermonkey

Tampermonkey 向けに開発された、X（旧 Twitter）のプロフィール「メディア」タブで投稿単位のメディアを収集・一括ダウンロードするユーザースクリプトです。今後の実装は「URL 収集 → メディア解析 → ZIP ダウンロード」の 3 段階に整理されています。

## 概要

- 対象ページ: `https://x.com/<screenName>/media`
- 主なフロー:
  1. プロフィール「メディア」タブを末尾まで自動スクロールし、メディア付きツイートの URL を一覧化
  2. 必要に応じて対象ツイートを再取得し、画像 (`pbs.twimg.com`) と動画 (`video.twimg.com`, m3u8 は除外) の直リンクを抽出
  3. 抽出したメディアを ZIP にまとめてダウンロード
- 目的: ツイート単位で欠けなくメディアを収集し、最終的に圧縮ファイルとして配布できるようにする

## 実装全体構成

- Tampermonkey ユーザースクリプトとして動作
- 主要コンポーネント（段階順）:
  - URL 監視（SPA 対応）とターゲットページ検知
  - 右上固定 UI パネル（収集／解析／ダウンロードボタン、ステータス、リスト表示）
  - 自動スクロールエンジン（実際にスクロール可能な要素を検出し、終端まで移動）
  - ツイート URL 収集ロジック（カード DOM からメディア付き投稿を抽出）
- メディア解析ロジック（必要なツイート HTML を再取得し、画像・動画 URL を網羅）
  - ZIP 生成およびダウンロード処理（検討中）

## デバッグモードについて

- パネルの「デバッグ」を有効にすると、メディア取得時に最初の 1 件のみを対象に HTML を記録します。
- 取得した HTML が `JavaScriptを使用できません。` のプレースホルダを示す場合、同じツイート詳細ページをバックグラウンドタブで自動的に開き、ブラウザ上で直接 DOM を確認できるようにします。
- デバッグ取得時は該当ポストを新しいタブでアクティブに開き（自動で閉じます）、`div[role="group"]` が画像／動画を含むまで（最大約 45 秒）待機してから HTML を拡張機能経由で受信し、パネルのデバッグ枠に自動表示します。
- Tampermonkey で `GM_openInTab` が利用できない場合は通常の新規タブを開き、フォーカスを元のタブへ戻すようにフォールバックします。

## 想定 UI フロー

- 表示条件: `/media` ページに遷移した場合にのみパネルを生成
- パネル構成:
  - ヘッダ: 「Media Tools」＋ドラッグ移動ハンドル
  - コントロールボタン
    - `URL収集`: 自動スクロールを実行し、メディア付きツイート URL を取得
    - `メディア取得`: 取得済みツイート URL を基に画像・動画 URL を解析
    - `ZIPダウンロード`: 抽出済みメディアをまとめてダウンロード（ZIP 生成）
    - `停止`: 進行中の処理をキャンセル
  - ステータス欄: 現在の処理段階、進捗件数、警告などを表示
  - リスト欄:
    - `URL収集` 完了時: ツイート URL の一覧を表示（クリックでツイートを開く）
    - `メディア取得` 完了時: `[IMG]` / `[MP4]` プレフィックス付きでメディアファイル名とリンクを表示

## 自動スクロール仕様（URL収集フェーズ）

- スクロールコンテナ検出: `document.scrollingElement`・`main`・`primaryColumn` などから最もスクロール可能な要素を優先的に選択
- スクロール終了条件:
  - `scrollHeight` が 3 ラウンド連続で増加しなくなる
  - または安全タイムアウト（90〜120 秒）に達する
- ステータス表示例: 高さの変化量、検出ツイート件数、idle カウンタ等

## ツイート URL 収集

- 対象セレクタ: `li[role="listitem"]` 直下のメディアカード（アイコンやサムネイルを含む）
- 抽出内容:
  - ツイート URL (`/screenName/status/tweetId/...`)
  - multi メディア判定 (カード内のマルチアイコンやメタ情報を利用)
  - スクリーンネーム、ツイート ID 等のメタ情報
- UI 表示: 収集済みツイート URL のリンクリストを生成

## メディア解析

- `メディア取得` ボタン押下で以下を実施:
  - `URL収集` 済みのツイートを対象に、必要な HTML をフェッチ
  - `<img>` の `src`、`background-image`、および `<video>/<source>` の `src` を抽出
  - 画像は `?name=orig`、`format` パラメータで正規化、動画は `.mp4` のみ対象
  - ツイート内での順序・種別をメタ情報として保持
- UI 表示: `[IMG] filename.jpg` / `[MP4] filename.mp4` の形式でリンクを表示し、クリックで元ファイルを開ける

## ZIP ダウンロード（計画）

- 解析済みメディアをまとめ、Blob ベースで ZIP アーカイブを生成
- 生成した ZIP に `screenName_media_<timestamp>.zip` などのファイル名を付けてダウンロード
- 大量メディア時のメモリ対策・進捗表示を追って検討

## ステータス表示

- URL 収集中: スクロール位置／検出ツイート数／idle 回数
- メディア解析中: 現在処理中のツイート番号や成功／失敗件数
- ZIP 生成中: 圧縮進捗・ダウンロード準備完了通知

## 制約・注意事項

- X 側 DOM / API の改変によりセレクタや挙動が変わる可能性がある
- m3u8・ライブ配信メディアは対象外（対応予定なし）
- 大量メディア取得時はブラウザメモリ使用量に注意（段階的な ZIP 生成が必要）
- 利用にあたり、Twitter (X) の利用規約および著作権を順守すること
